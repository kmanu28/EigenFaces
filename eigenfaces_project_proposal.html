01 — Problem Statement

What are we building?

Given a database of known faces, can we identify who a new face belongs to — using only linear algebra, no machine learning?

Each face image is a grid of pixel values. A 64×64 image is a vector of 4096 numbers. The key insight of Eigenfaces is that all human faces don't randomly fill this 4096-dimensional space — they cluster in a small subspace. We find that subspace using eigendecomposition, project every face into it, and identify unknown faces by finding the closest match. Pure linear algebra, real results.

02 — Linear Algebra Pipeline

Step-by-step workflow

Every step of the prescribed pipeline is used and naturally motivated by the face recognition problem.

Step 01 · Matrix Representation
Faces as Vectors
Load 400 face images. Flatten each 64×64 image into a 4096-dimensional vector. Stack all 400 vectors as rows to form the data matrix A (400 × 4096). Each row is one face. The matrix represents the entire face database.
Matrices Linear Transformations Systems of Equations
Step 02 · Matrix Simplification
Mean-Centering & RREF
Subtract the mean face from every face vector to center the data. Apply RREF to understand the rank of the face matrix — how many truly independent facial patterns exist among 400 images. This reveals the effective dimensionality of face space.
Gaussian Elimination RREF
Step 03 · Structure of the Space
Face Space Analysis
Compute the rank and nullity of the face matrix. The column space defines "face space" — the subspace that all valid face images live in. The null space contains pixel patterns that carry no facial information. We establish that real faces occupy a tiny corner of the full pixel space.
Vector Spaces Subspaces Rank & Nullity Column Space
Step 04 · Remove Redundancy
Independent Facial Patterns
Many faces share similar features (two eyes, a nose, a mouth). Extract a linearly independent basis from the face vectors, removing redundant faces that don't add new information. This reduces 400 faces to a compact set of independent basis faces.
Linear Independence Basis Selection
Step 05 · Orthogonalization
Gram–Schmidt on Face Basis
Apply Gram–Schmidt orthogonalization to the independent face basis vectors to produce orthogonal "pure facial direction" vectors. Orthogonality ensures that when we project a face onto this basis, each component measures a truly independent facial feature.
Orthogonal Vectors Gram–Schmidt Orthogonal Bases
Step 06 · Projection
Project Faces onto Eigenface Space
Project every face (training and test) onto the orthogonal eigenface subspace. Each face is now represented as a short coordinate vector (weights) in eigenface space instead of 4096 pixel values. This is the key compression step — faces become small, comparable fingerprints.
Orthogonal Projections Projection onto Subspaces
Step 07 · Prediction via Least Squares
Identify Unknown Faces
To recognize a test face, solve the least squares problem: find the training face whose projection minimizes the squared distance to the test face's projection. Use x̂ = (AᵀA)⁻¹Aᵀb to find the best match. The person with minimum reconstruction error is the predicted identity.
Least Squares Solution Matrix Transpose & Inverse
Step 08 · Pattern Discovery
Eigenfaces via Covariance Matrix
Compute the covariance matrix C = AᵀA of the mean-centered face matrix. Find its eigenvalues and eigenvectors. The eigenvectors are the Eigenfaces — ghostly face-shaped images that represent the dominant modes of variation across all faces in the database. Large eigenvalues = more important facial features.
Eigenvalues Eigenvectors Covariance Matrix
Step 09 · System Simplification
Diagonalization & Dimensionality Reduction
Diagonalize the covariance matrix as C = PDP⁻¹. Keep only the top-k eigenvectors (typically k=50 out of 4096). Show via a scree plot that these k components capture 95%+ of variance. The system is reduced from 4096 dimensions to k — same recognition accuracy, fraction of the computation.
Diagonalization Symmetric Matrix Decomposition

04 — Viva Preparation

Concept → Purpose → Outcome

For every concept used, be ready to answer all three. Here are the exact answers.

Matrix Representation
WhatEach 64×64 face image is flattened into a 4096-dimensional vector. All 400 vectors are stacked into matrix A.
WhyLinear algebra operates on matrices. Representing faces as vectors lets us apply all matrix operations to face data.
OutcomeA 400 × 4096 matrix where each row is a complete face — ready for decomposition.
RREF & Rank Analysis
WhatRow reduce the face matrix to find its rank — how many linearly independent face patterns exist.
Why400 faces don't all carry unique information. Rank tells us the true dimensionality of the face space.
OutcomeConfirmed that the effective rank is much lower than 400 — many faces are linearly dependent combinations of others.
Gram–Schmidt Orthogonalization
WhatConvert the independent face basis vectors into a set of orthogonal vectors.
WhyOrthogonal bases make projection simple and exact — each eigenface captures a completely independent aspect of facial variation.
OutcomeAn orthogonal set of basis vectors that span face space — a clean coordinate system for all faces.
Eigenvalues & Eigenvectors (Eigenfaces)
WhatEigendecomposition of the covariance matrix AᵀA. Each eigenvector, reshaped to 64×64, is an Eigenface.
WhyThe eigenvectors point in the directions of maximum variance across all faces — they capture what makes faces different from each other.
OutcomeThe top eigenfaces (ghostly face images) that together form a complete basis for the face space.
Orthogonal Projection
WhatProject each face onto the eigenface subspace spanned by the top-k eigenvectors.
WhyThe projection gives a compact k-dimensional representation (weights) of every face. Comparing weights is far easier than comparing 4096 pixels.
OutcomeEvery face is now a short vector of k weights — its "face fingerprint" in eigenspace.
Least Squares for Recognition
WhatFind the training face that minimizes the squared distance to a test face's eigenspace projection.
WhyRecognition is an overdetermined system — we use least squares to find the best match across all training samples.
OutcomePredicted identity of the test face with a confidence score based on reconstruction error.
Diagonalization
WhatExpress the covariance matrix as C = PDP⁻¹ where D is diagonal (eigenvalues) and P contains eigenvectors.
WhyDiagonalization lets us rank eigenvalues and select only the top-k, compressing the model from 4096 to ~50 dimensions with minimal information loss.
OutcomeA compressed face recognition model that uses 50 dimensions instead of 4096, with 90%+ recognition accuracy.
05 — Dataset & Setup

Zero setup required

The dataset is built directly into scikit-learn — no downloading, no Kaggle account, no file management. One function call loads all 400 face images, pre-labelled and ready to use.

y = data.target      # labels: 0–39 (40 subjects, 10 images each)

# Flatten to matrix form
import numpy as np
